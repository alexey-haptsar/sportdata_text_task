# SportdataTestApp — VPN-клиент для macOS

Переработанный интерфейс VPN-клиента. SwiftUI, MVVM, macOS 15.

## Как запустить

1. Нужен macOS 15+ и Xcode 16+
2. Открыть `SportdataTestApp/SportdataTestApp.xcodeproj`
3. Выбрать схему `SportdataTestApp`, destination — `My Mac`
4. `Cmd + R`

Никаких зависимостей нет, всё собирается из коробки.

## Какие проблемы UI были исправлены

В исходном макете было несколько вещей, которые бросались в глаза:

- **Огромная зелёная кнопка Connect** — она занимала слишком много места и выглядела чужеродно для macOS. Я заменил её на стандартную `.borderedProminent` кнопку с `.controlSize(.large)` — она всё ещё главный элемент на экране, но не выбивается из общей стилистики системы.
- **Спиннер загрузки висел отдельно** — в оригинале он был визуально оторван от статуса подключения, непонятно к чему относился. Сейчас нативный `ProgressView` встроен прямо в индикатор статуса и появляется вместо иконки, когда идёт подключение.
- **Список серверов не выглядел нативно** — заменил на стандартный macOS `Picker` в стиле `.menu` (выпадающий список). Добавил флаги стран, чтобы серверы можно было различать не только по тексту.
- **Не было визуального разделения между блоками** — всё сливалось в одну кучу. Добавил разделители и нормальные отступы, чтобы чётко читались три зоны: статус, выбор сервера, кнопка действия.
- **Статус был просто текстом** — непонятно, хорошо это или плохо. Добавил цветовую индикацию и иконки (`shield.slash` / `checkmark.shield.fill`), плюс пояснительный текст под статусом ("Not Protected", "Your Connection is Secure" и т.д.).
- **Settings и Help болтались без логики** — перенёс их вниз экрана как второстепенные элементы, чтобы не отвлекали от основного действия.

## Какие UX-решения были приняты

- **Компактное окно фиксированного размера (320×420)** — VPN-клиент это утилита, ему не нужно большое окно. Все элементы видны сразу, ничего лишнего.
- **Цветовое кодирование состояний** — красный (отключён), оранжевый (подключение), зелёный (подключён). Цвет меняется и на индикаторе, и на кнопке одновременно — состояние считывается моментально.
- **Выбор сервера блокируется при активном подключении** — чтобы нельзя было случайно переключить сервер во время сессии. Picker разблокируется только после отключения.
- **Кнопка отражает текущее состояние** — меняется надпись ("Connect" → "Connecting…" → "Disconnect"), цвет и доступность. Пользователь всегда понимает, что произойдёт по нажатию.
- **Только светлая тема** — зафиксировал через `.preferredColorScheme(.light)`, чтобы не было сюрпризов с цветами.
- **Settings и Help открываются в поповерах** — пользователь не уходит с главного экрана и не теряет контекст текущего подключения.
- **Окно нельзя ресайзить** — интерфейс спроектирован под конкретный размер, растягивание только сломает вёрстку.

## Структура проекта

```
SportdataTestApp/
├── SportdataTestAppApp.swift              — Точка входа
├── Models/
│   ├── VPNConnectionState.swift           — Enum состояний подключения
│   └── VPNServerInfo.swift                — Модель сервера
├── ViewModels/
│   └── VPNConnectionViewModel.swift       — Логика подключения (заглушка с задержкой 2 сек)
├── Views/
│   ├── VPNDashboardView.swift             — Главный экран
│   ├── ConnectionStatusIndicator.swift    — Индикатор статуса
│   ├── ServerSelectionPanel.swift         — Выбор сервера
│   ├── ConnectionToggleButton.swift       — Кнопка подключения
│   ├── SettingsContentView.swift          — Поповер настроек
│   └── HelpContentView.swift             — Поповер помощи
└── Assets.xcassets/                       — Ресурсы
```

Архитектура — **MVVM**. View подписываются на ViewModel через `@StateObject` / `@Published`, модели — обычные value types.
